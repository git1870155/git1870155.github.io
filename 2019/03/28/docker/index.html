docker是什么？解决了什么问题

Docker是一个开源的引擎，可以轻松的为任何应用创建一个轻量级的、可移植的、自给自足的容器。通俗的来说docker就是存放应用的一个容器 , ==*在docker集装箱之前，货物(应用程序)运输没有统一的标准方式进行搬运，于是铁路(开发)、公路(测试)、海洋(运维)等各种运输之间，需要大量的人力作为货物中转(解决环境问题)，效率极低，而且成本很高。集装箱出现之后，世界上绝大多数的货物运输都可以放到这个神奇的箱子里，然后在公路、铁路、海洋等所有运输场景下，这个箱子都可以不用变化形态直接可以承运，而且中间的中转工作，都可以通过大型机械搞定，效率大大提升。

解决了什么问题
  • 我的机子测试没事啊,一到你这怎么全是错误
  • 线下已经测了好几遍,一上线就出问题
  • 大促都快到了,扩容服务器环境还没搞好
  • 等等
优势
  • 轻量级 开销极低 更高效的资源利用
  • 启动在毫秒级别
  • 安全,隔离性高 运行在不同容器的服务互不影响
  • 移植性高 跨平台
  • 更新方便 使用dockerfile迭代升级镜像 只需简单几行配置,代替大量手工操作
  • 快速部署 可做到持续交付

对于docker的几个理解

镜像 与 容器

镜像是文件, 类似于虚拟机镜像 ,是一个只读的文档, 容器是进程。 容器是基于镜像创建的 , 也就是在镜像之上增加一层可读层 , 即容器中的进程依赖于镜像中的文件 , 这里的文件包括进程运行所需要的可执行文件， 依赖软件， 库文件， 配置文件等等…

仓库

镜像的管理 存放地方

公有仓库

docker hub docker官方维护的一个公共仓库 ,目前大约有15000多个镜像(服务器在国外访问慢) 阿里云仓库(也有免费创建私有仓库) 提供大量的镜像下载 用户也可以推送

私有仓库

比如说是公司内部搭建的仓库或者个人搭建 不对外

docker的服务管理

docker-machine

Docker 官方提供的一个工具，它可以帮助我们在远程的机器上安装 Docker，或者在虚拟机 host 上直接安装虚拟机并在虚拟机中安装 Docker。我们还可以通过 docker-machine 命令来管理这些虚拟机和 Docker 提供了docker多平台集中管理

dokcer-compose

是一个定义及运行多个Docker容器的工具。使用Docker Compose你只需要在一个配置文件中定义多个Docker容器，然后使用一条命令将多个容器启动，Docker Compose会通过解析容器件的依赖关系（link, 网络容器 –net-from或数据容器 –volume-from）按先后顺序启动所定义的容器。

docker 网络模式
bridge模式
bridge模式 默认docker使用的模式 类似于vmware中的nat模式 在该模式下docker守护进程会创建虚拟的以太网网桥(docker0),对宿主机与容器之间进行网络数据包转发,也就是同一个宿主机下的所有容器在同一网段 ,相互之间可以通信

host模式
host模式 –net=host 容器直接使用宿主机网络ip 这种方式把容器直接暴露在公网之中

container模式
container模式 –net=container:name 容器使用其他容器的网络命名空间 也就是说容器之间使用相同的网络

none模式
none模式 , –net=none 关闭了容器的网络功能 比如说数据卷容器(容器不需要网络通信), 或者说你需要自己定义网络

docker 数据卷 与 数据卷容器

数据卷
类似于linux mount挂载操作
挂载一个目录作为数据卷 也可挂载一个文件作为数据卷

数据卷容器
数据卷容器其实就是一个容器,专门用它提供数据供其他容器使用
例如 创建数据卷容器db 另外两个容器使用–volumes-from参数挂载,在数据卷容器中创建1.txt文件 同时另外两个容器都可读取到

开发环境使用docker优势
可以让部门开发环境一致,且线上线下环境一致,避免由于个人自己搭建的环境,由于版本不一致,操作系统平台不一致,软件版本 扩展库等小问题影响到开发效率 ,产生的在我这运行好好的,到你这怎么全是bug的问题,而且在团队新成员新加入时也不用环境问题而困扰,环境迭代方便,一次更新 只需拉去增量的docker镜像即可,大大提升团队协作效率,方便开发.测试.上线
